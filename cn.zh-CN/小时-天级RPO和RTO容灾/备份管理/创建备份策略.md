# 创建备份策略 {#concept_71521_zh .concept}

备份策略是确定备份内容，备份时间，备份方式的核心。

一个备份策略需要包含以下4个参数：

```
- 全量备份的时间和频率
- 增量（或者差异）备份的时间和频率
- 备份是否上阿里云
- 本地备份和云上备份的保留时间

```

## 备份策略的创建流程 { .section}

1.  在主菜单中单击**备份策略**进入备份策略页面。

2.  单击**创建**，创建备份策略。

    ![开始创建策略](http://docs-aliyun.cn-hangzhou.oss.aliyun-inc.com/assets/pic/61847/cn_zh/1512044971475/%E5%B1%8F%E5%B9%95%E5%BF%AB%E7%85%A7%202017-11-29%20%E4%B8%8B%E5%8D%888.59.47.png)


## 示例 { .section}

下面的例子演示了如何配置一个典型的备份策略。

1.  制定备份策略：

    -   每周六晚上12点做一次全量备份
    -   周日到周五晚上每天做一次增量备份
    -   备份数据上传阿里云。
    -   本地备份保留14天时间，阿里云上备份保留90天。
2.  创建备份策略：

    -   创建全量备份策略
    ![策略-全量](http://docs-aliyun.cn-hangzhou.oss.aliyun-inc.com/assets/pic/61847/cn_zh/1512045099035/%E5%B1%8F%E5%B9%95%E5%BF%AB%E7%85%A7%202017-11-29%20%E4%B8%8B%E5%8D%889.01.13.png)

    -   在同一个对话框中添加一个增量备份策略
    ![策略-增量](http://docs-aliyun.cn-hangzhou.oss.aliyun-inc.com/assets/pic/61847/cn_zh/1512045140411/%E5%B1%8F%E5%B9%95%E5%BF%AB%E7%85%A7%202017-11-29%20%E4%B8%8B%E5%8D%889.01.51.png)

3.  策略创建好之后，您就可以基于这个策略，给被保护的服务器添加一个备份计划。
